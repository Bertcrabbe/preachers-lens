# Architecture

Preacher's Lens is a single-page application built with React 18 + Vite. The backend is Supabase (PostgreSQL database, built-in auth, file storage, and Deno edge functions). AI features run through Supabase edge functions that call AssemblyAI for transcription, the Lovable AI Gateway (Gemini models) for analysis, and ElevenLabs for voice synthesis.

## Directory Structure

```
preachers-lens/
├── src/
│   ├── components/           # UI components (shadcn/ui primitives + custom)
│   │   └── ui/               # shadcn/ui primitive components
│   ├── pages/                # Route-level page components
│   ├── hooks/                # Custom React hooks
│   ├── utils/                # Utility functions and Web Workers
│   ├── contexts/             # React Context providers (Theme, Auth)
│   ├── integrations/
│   │   └── supabase/         # Supabase client + auto-generated TypeScript types
│   └── lib/                  # Shared helpers (cn utility)
├── supabase/
│   ├── functions/            # 25 Deno edge functions
│   └── migrations/           # SQL migration history
├── scripts/                  # Harness scripts (risk, docs-drift, UI capture)
└── docs/                     # Project documentation
```

## Data Flow

```
Browser → Supabase JS client → Supabase PostgreSQL → real-time subscription update
Browser → fetch() → Supabase edge function → AssemblyAI / Lovable AI Gateway / ElevenLabs
```

Components read and write the database directly through the Supabase JS client. Long-running AI jobs (transcription, analysis) are kicked off via edge function calls; the frontend polls or subscribes to status changes on the `sermons` table.

## Auth Flow

```
Browser → Supabase Auth (email/password) → JWT stored in localStorage → auto-refresh
```

`src/integrations/supabase/client.ts` initialises the Supabase client with `persistSession: true` and `autoRefreshToken: true`. A global auth state listener redirects unauthenticated users to `/auth`. All database queries include the user's JWT automatically via the client, which Supabase uses to enforce row-level security.

## Key Invariants

- **`src/integrations/supabase/types.ts` is auto-generated.** It is regenerated by the Supabase CLI from the live database schema. Never edit it by hand — changes will be overwritten.
- **All queries are user-scoped.** Every table has a `user_id` column. Queries always filter by the authenticated user's ID to prevent cross-user data leakage.
- **Edge functions are Deno, not Node.js.** Import from `npm:` or `https://deno.land/` inside `supabase/functions/`. Node.js built-ins and CommonJS `require()` are not available.
- **AI jobs are fire-and-poll.** Edge functions that call external AI APIs return immediately (202) and write status back to the `sermons` or `sermon_metrics` table. The frontend subscribes to those rows for updates.
